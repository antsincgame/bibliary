# /commit — Автокоммит

Сделай коммит текущих изменений с осмысленным сообщением на русском.

## Workflow

Запусти параллельно:
- `git status` (без `-uall`)
- `git diff HEAD`
- `git log --oneline -5` (чтобы стиль сообщений соответствовал репо)

Затем:
1. Проанализируй diff — что реально изменилось и **зачем**
2. Составь сообщение:
   - **Первая строка** — императив, ≤72 символа, на русском («добавить X», «починить Y», «отрефакторить Z»)
   - **Тело** (опционально) — почему, если из diff не очевидно
3. Стейдж явных файлов (`git add path1 path2`), **не** `git add -A` / `.`
4. `git commit -m "..."` через HEREDOC

## Запреты

- Не коммитить `.env`, `*.pem`, `credentials*`, `secrets*`, `*.key`
- Не использовать `--amend` без явной просьбы (амендят предыдущий коммит, не текущий)
- Не пушить (`git push` — отдельная команда, спрашивай отдельно)
- Не пропускать хуки (`--no-verify`) — если хук падает, чини причину

## Пример хорошего сообщения

```
добавить language-aware crystallizer для украинских книг

prefs.ukrainianSpecialistModel теперь используется через
resolveCrystallizerForLanguage в extraction-runner. Fallback на
обычный crystallizer если specialist не сконфигурирован.
```

## Пример плохого

```
fix
update
изменения
WIP
```

Не используй такие.
