# /ui-tester — Проверка UI

Сканируй UI, выяви мёртвые элементы. Двунаправленно: и UI без логики, и логика без UI.

## Что сканируется

| Элемент | Что должно быть |
|---------|----------------|
| `Pressable`, `TouchableOpacity`, `Button` | `onPress` с реальной функцией |
| `Switch`, `Checkbox`, `RadioGroup` | `onValueChange` |
| `TextInput` | `onChangeText`, `onSubmitEditing` |
| `FlatList`, `ScrollView`, `SectionList` | `onRefresh`, `onEndReached` если нужны |
| `Link`, `router.push`, `navigate` | Маршрут существует в навигации |
| `<form>`, `<button type="submit">` | `onSubmit` реальный |

## Классификация обработчиков

### ❌ Мёртвые (CRITICAL)
- `onPress={() => {}}`
- `onPress={() => console.log('TODO')}`
- `onPress={() => Alert.alert('Coming soon')}`
- `onPress={() => {}}` с комментарием `// TODO`

### ⚠️ Подозрительные (WARNING)
- Маршрут на `router.push('/screen')` где `/screen` не существует
- Store action который ничего не делает (пустое тело)
- Обработчик-обёртка над таким же мёртвым

### ✅ Рабочие
- Реальная навигация на существующий экран
- Zustand / Redux action с настоящей логикой
- API call с обработкой ответа

## Фаза 5.5 — Двунаправленная проверка (КРИТИЧНО)

Не достаточно проверить только UI → логика. Нужны три направления:

- **Случай A**: функция есть в store/контроллере — UI её не зовёт. **Мёртвая логика.**
- **Случай B**: UI зовёт функцию которой нет / она пустышка. **Мёртвый UI.**
- **Случай C**: парная асимметрия. `addItem` есть и в store и в UI, но `removeItem` есть только в store — UI забыли.

## Workflow

1. Grep по UI-примитивам в области
2. Для каждого обработчика — Read функции, на которую ссылается
3. Параллельно — Grep по store/контроллеру: какие экшены вообще есть
4. Сверь: UI ↔ store. Зафиксируй мёртвых и асимметрии
5. Выдай отчёт по категориям с file:line ссылками
