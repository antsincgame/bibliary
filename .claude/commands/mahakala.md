# /mahakala — Защита от регрессий

Активирован защитный режим. Любая правка проходит 4 фазы. Если правка ломает базовый снимок — блокируй до явного «всё равно делай».

## Алгоритм

### Фаза 1: Базовый снимок (ДО правки)
Запусти параллельно:
- `tsc --noEmit` (или эквивалент для языка проекта)
- линтер (`npm run lint`, `ruff`, etc.)
- быстрые тесты
- build

Зафиксируй цифры: сколько ошибок, какие предупреждения, статус билда. Это baseline.

### Фаза 2: Карта влияния
Найди:
- Кто импортирует затрагиваемый файл/символ
- Кто использует затрагиваемый тип/интерфейс
- Какие тесты покрывают эту область
- Внешние контракты (публичные API, файлы конфига, миграции)

Используй Grep / Agent(subagent_type=Explore).

### Фаза 3: Оценка риска

| Уровень | Критерии | Действие |
|---------|----------|----------|
| 🟢 НИЗКИЙ | Изолированно, <5 строк, не публичный API | Правка + быстрая проверка |
| 🟡 СРЕДНИЙ | 1-3 зависимых файла | Правка + проверка всех зависимых |
| 🔴 ВЫСОКИЙ | 4+ зависимых / публичный API / тип | Пошаговая правка с промежуточными проверками |
| ⚫ КРИТИЧЕСКИЙ | Ядро системы, миграции, security | Сначала план + явное одобрение |

### Фаза 4: Постпроверка (ПОСЛЕ правки)
Запусти те же команды, что в фазе 1. Цифры должны быть **≥ baseline** (или явное объяснение почему хуже допустимо). Если хуже — откатываемся или останавливаемся для разговора.

## Запреты

- Не править наугад «авось не сломается»
- Не запускать только тесты которые удобны — bаseline должен быть полным
- Не подавлять ошибки через `// @ts-ignore` без записи причины

Правка для анализа: $ARGUMENTS
