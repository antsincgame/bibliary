# /chainlogic — Гроссмейстер цепочек

Спланируй оптимальную последовательность действий через HTN-декомпозицию, Tree-of-Thoughts и A*-эвристику.

## Уровни детализации (выбираются автоматически)

| Уровень | Когда применять | Глубина |
|---------|-----------------|---------|
| **CHAIN-LITE** | ≤5 шагов, 1 файл, низкий риск | 2 альтернативные цепочки |
| **CHAIN-STANDARD** | 5-15 шагов, несколько файлов | DAG зависимостей + 3 цепочки |
| **CHAIN-DEEP** | 15+ шагов, высокая цена ошибки | Все 6 фаз алгоритма |

## Подрежимы

- `/chainlogic plan` — только план, не исполнять
- `/chainlogic execute` — план + исполнение через TodoWrite
- `/chainlogic compare` — сравнить 2-3 стратегии, выдать таблицу trade-off
- `/chainlogic research` — внешний research (WebSearch / Perplexity если доступен) перед планом
- `/chainlogic recover` — задача застряла, спланируй обход

## Алгоритм (6 фаз для DEEP)

### 1. Library Lookup
Проверь, существует ли готовое решение в проекте / в стандартных либах / в документации. Не изобретать велосипед.

### 2. Декомпозиция (HTN)
От верхнего уровня задачи к атомарным действиям. Дерево, не плоский список.

### 3. Граф зависимостей (DAG)
- Какие шаги обязаны идти последовательно (есть data dependency)?
- Какие можно параллелить?
- Где критический путь (самая длинная цепочка)?

### 4. Альтернативные цепочки (Tree-of-Thoughts)
Минимум **2-3 разных пути** к цели. Не просто переставленные шаги — реально разные стратегии.

### 5. Оценка (A* эвристика)
Для каждого пути:

| Метрика | Что считать |
|---------|------------|
| Cost | время / токены / нагрузка |
| Risk | вероятность поломки |
| Reversibility | насколько легко откатить |
| Side-effects | что побочно меняется |
| Info-value | сколько узнаём по дороге |

### 6. Исполнение (ReAct + Reflexion)
**Reason → Act → Observe → Reflect** на каждом шаге. После Reflect — обнови план если выяснилось новое.
