# /search — Веб-поиск

Найди информацию в интернете по запросу пользователя. Используй WebSearch для быстрого поиска и WebFetch если нужно прочитать конкретную страницу целиком.

## Когда использовать

- Документация библиотеки / API
- Свежие changelogs и breaking changes
- Решения по конкретным ошибкам / стек-трейсам
- Бенчмарки и сравнения подходов
- Текущие best practices в области

## Когда **не** использовать

- Внутренняя кодовая база → используй Grep / Read / Agent(subagent_type=Explore)
- Известные базовые знания → отвечай сразу из своей памяти
- Запросы которые требуют свежих данных, но имеют ответ в `.claude/rules/` → читай локально

## Workflow

1. **WebSearch** с фокусированным запросом (не «как работает React», а «React 19 useOptimistic breaking changes»)
2. Если результаты дают конкретный URL с глубоким ответом — **WebFetch** этого URL для деталей
3. Дай ответ с **источниками** — каждое утверждение должно иметь ссылку
4. Если результаты противоречат друг другу — явно покажи противоречие, дай свою оценку

## Запреты

- Не выдумывать URL «которые наверное существуют»
- Не выдавать гипотезу из памяти за факт из поиска
- Не пересказывать первый результат как истину — кросс-чек ≥2 источника на важных утверждениях

## Альтернатива через Perplexity MCP

Если в окружении доступен Perplexity MCP (`mcp__perplexity__search` или подобный), используй его для **deep research** запросов — он лучше синтезирует. Для быстрых лукапов WebSearch достаточно.
