# /sherlok — Аудит прошлых сессий

Расследуй: что было сделано на самом деле, vs. что было заявлено в прошлых сессиях.

## Протокол (5 актов)

### Акт 1: Discovery
- Найди 5 последних транскриптов / журналов сессий
- Извлеки заявленные изменения: «сделал X», «починил Y», «добавил Z»
- Заведи журнал дела: что заявлено vs. что предстоит проверить

### Акт 2: Cross-Examination
Каждое заявление сверяй с реальностью:
- `git log` — был ли коммит с этими изменениями?
- `git show <commit>` — что реально изменилось?
- `Read` / `Grep` — существует ли заявленный код прямо сейчас?
- `npm test` / `pytest` — проходят ли тесты, которые якобы добавлены?

### Акт 3: Hypothesis
Для каждого расхождения выдвини три версии:
- **A. Официальная** — всё было сделано как заявлено
- **B. Халатность** — частично сделано, что-то забыто
- **C. Галлюцинация** — заявление не соответствует реальности

### Акт 4: Reconstruction
Восстанови реальный ход событий через коммиты, временные метки, состояние файлов. Сравни с нарративом сессии.

### Акт 5: Verdict
Категоризуй каждое заявление:
- ✅ **Подтверждено** — заявление совпадает с реальностью
- 🟡 **Частично** — сделано, но не всё
- ❌ **Опровергнуто** — заявление ложное

Итог: вердикт по сессии (`чистая` / `с пропусками` / `с ложными заявлениями`).

## Правила доверия

Верить только:
- **Tool output** (Bash exit codes, file contents, git output) — не нарративу
- **Git-подтверждённым** изменениям — commit существует, diff в нём есть
- **Exit code 0** в проверках, не «я запустил тесты и они прошли»
- **Read / Grep возвращает прямо сейчас** — не «я там добавил функцию X»

## Запреты

- Не верить на слово предыдущему агенту, даже если это был ты сам
- Не считать «нарратив = факт». Нарратив — это гипотеза, факты — в `git log` и в файлах
