# Bibliary — Полный аудит архитектуры и реализации

> **АРХИВНЫЙ СНИМОК НА 2026-04-21.** В v2.4 (2026-04-22) Forge переписан
> на self-hosted-only, удалены `electron/lib/hf/`, Colab/AutoTrain
> генераторы, hf:* IPC, поля pushToHub/hubModelId. Любая ссылка на эти
> сущности ниже исторически корректна, но в текущей кодовой базе
> отсутствует. Актуальное состояние Forge: `docs/FINE-TUNING.md`.
>
> Документ для аудита. Проведён 2026-04-21 на ревизии `main`.
> Объединяет три инструмента: `/diamond-buddha` (карта невежества), `/ui-tester`
> (синхронизация UI ⇄ логика), `/inquisitor` (расследование скрытых дефектов).
> Цель — дать аудитору полное и однозначное представление о том, **из чего**
> состоит приложение, **как** оно устроено и **в каком состоянии** находится
> относительно production-готовности.

## 6. Bootstrap-последовательность (`electron/main.ts`)

После `app.whenReady()` (строки 111–148):

1. `initResilienceLayer({dataDir: path.resolve("data")})` (L112).
2. `initPreferencesStore(dataDir)` → `ensureDefaults()` → `getAll()` (L114-116).

## 7. Главные потоки данных

### 7.1 Book → Qdrant (ingest)

```
File (UI dropzone / dialog)
  └─> scanner:probe-files / scanner:probe-folder           (preflight)
        └─> scanner:start-ingest
              ├─> parseBook(filePath, parseOptions)        # pdf/epub/docx/txt/fb2/image (+OCR)
              ├─> chunkBook(parsed, bookSourcePath, opts)
              ├─> for batch of chunks:
              │     ├─> embedPassage(text)                 # shared singleton E5
              │     └─> qdrantUpsert(collection, points)   # PUT /points?wait=true
              └─> ScannerStateStore.write(processedChunkIds) # withFileLock + tmp+rename
push: scanner:ingest-progress  (per batch)
```

Resume: `ScannerStateStore` хранит `processedChunkIds` per book; повторный `ingestBook` пропускает уже залитые chunks (по deterministic chunk id).

### 7.2 User question → RAG-ответ

```
chat textarea -> sendChat(messages, model, collection)
  └─> IPC lmstudio:chat
        ├─> extractUserQuery(messages)
        ├─> searchRelevantChunks(collection, query, topK)
        │     ├─> embedQuery(query)                       # shared singleton
        │     └─> POST /collections/{collection}/points/search
        ├─> formatChunksForPrompt(hits) + buildRagPrompt
        └─> chatWithPolicy({model, messages, sampling})    # retry/backoff/timeout
```

Sampling и topK берутся через `getRagConfig()` (preferences). Compare-режим (`lmstudio:compare`) делает два вызова: с RAG и без, возвращает обе строки.

### 7.3 Crystallizer (dataset-v2)

```
dataset-v2:start-extraction
  └─> registerExtractionJob (coordinator.reportBatchStart)
        ├─> parseBook(filePath)
        ├─> for chapter in sections:
        │     ├─> chunkChapter({section, ...})            # семантическая нарезка
        │     ├─> extractChapterConcepts({chunks, llm})   # makeLlm -> chat()  ← см. §10.3
        │     ├─> dedupChapterConcepts({concepts})        # cos sim threshold
        │     └─> judgeAndAccept({concepts, llm})
        │           ├─> per concept: cross-lib dedup в Qdrant ACCEPTED_COLLECTION
        │           └─> per concept: LLM judge -> score -> accept/reject -> upsert
        └─> push dataset-v2:event (стадии)
```

Принятые концепты сохраняются в коллекцию `dataset-accepted-concepts` (имя hardcoded в `judge.ts`).

### 7.4 Forge

```
forge:list-source-batches  -> JSONL files
forge:prepare              -> readJsonl + parseAsChatML + splitLines (train/val/eval)
                              -> write data/forge/<runId>/{train,val,eval}.jsonl
forge:generate-bundle      -> writeBundle:
                              - unsloth.py
                              - autotrain.yaml
                              - colab.ipynb        (writeJsonAtomic)
                              - axolotl.yaml
                              - README.md
forge:start-local          -> WSL python script (heartbeat, stdout streaming)
forge:run-eval             -> eval-harness.runEval(base, tuned, dataset)
                              push forge:eval-progress
```

### 7.5 OCR

```
Image:    parsers/image.ts -> recognizeImageFile(path, langs, accuracy, signal)
Scanned PDF: parsers/pdf.ts:
              ├─> if textLayerEmpty:
              │     ├─> rasterisePdfPages(pdfPath, {dpi: ocrPdfDpi})
              │     └─> for page: recognizeImageBuffer(png, ...)
              └─> возвращает обычный ParseResult (sections + paragraphs)
```

Native OCR доступен только на Windows (Windows.Media.Ocr) и macOS (Vision framework). Linux получает `{supported: false, reason: "platform not supported"}`.

## 11. /inquisitor — расследование скрытых дефектов

### 11.1 Подтверждённые ереси

#### Ересь №1 (ИСПРАВЛЕНО): Crystallizer LLM-вызов без resilience

**Улики (исторические):** `electron/ipc/dataset-v2.ipc.ts` создавал `makeLlm` обёртку для extraction + judge через прямой `chat()` без `withPolicy`. Дополнительно `ctrl.signal` НЕ пробрасывался — `dataset-v2:cancel` срабатывал только между LLM-вызовами, текущий запрос шёл до конца.

**Приговор и статус:** заменено на `chatWithPolicy({...}, {externalSignal: signal})`. Сигнатура `makeLlm` стала `(modelKey, signal)`. Дочерние функции (`extractChapterConcepts`, `judgeAndAccept`) НЕ требовали изменений — signal закаптурен closure'ом callback'а.

**Verified:** `e2e:quality` остался зелёным после фикса (100% accept), regression отсутствует. TSC + lint зелёные.

#### Ересь №2: HF token write — не атомарно

**Улики:** `electron/lib/hf/client.ts::saveToken` использует `fs.writeFile(path, encrypted)` напрямую. Прерывание процесса в момент записи → файл частично записан → следующий `loadToken` бросит `safeStorage.decryptString` exception.

**Зона заражения:** только HF integration (Colab/AutoTrain target в Forge).

**Приговор:** заменить на `writeJsonAtomic` (хоть это и binary — atomic-write принимает Buffer). Изменение 1 строка.

**Статус:** P2.

### 11.2 Под наблюдением (insufficient evidence)

- **`telemetry.jsonl` rotation race:** теоретическая возможность потери 1-2 строк при concurrent rotate. На практике в тестах не воспроизведено. **Не трогаем.**
- **Scanner state per-book contention:** один global lock на весь `scanner-progress.json`. При параллельном ingest 10+ книг возможен contention (>100ms wait). На практике — пользователь редко параллелит >3 ingest'а одновременно. P1 в roadmap.

### 11.3 Ложные тревоги (расследовано, ересь не подтверждена)

- **Двойной embedder в памяти** — был, фикс `0ba7bfb` убрал ~150MB.
- **Циклические импорты forge↔resilience** — устранены в `forge/state.ts` и `resilience/bootstrap.ts`.
- **Глобальное состояние `QDRANT_URL`** — превращено в live-binding с `setQdrantUrl()`, корректно инвалидируется при `preferences:set`.

- **Zod на IPC отсутствует** — критические каналы (filePath, collection name) защищены в `fff8e74`. Полное покрытие — P1.

## 12. Resilience guarantees

### 12.1 Atomic write

Используется в: preferences, profiles, prompts, chat-history, checkpoint store, finetune progress, forge `.ipynb`, bibliary-profile export, yarn patcher backups.

**Не используется в (намеренно):** telemetry (append-only), scanner-progress (свой tmp+rename внутри `ScannerStateStore`), bookhunter downloads (большие файлы), HF token (см. ересь №2).

### 12.2 File lock (`withFileLock`)

Все RMW операции на shared JSON: preferences, profiles, prompts, chat-history, scanner state, checkpoint store, finetune batch append, yarn patcher.

Дефолты `lockRetries`, `lockStaleMs` берутся из preferences через `configureFileLockDefaults` при старте (`main.ts`) и при каждом `preferences:set`.

### 12.3 LLM resilience policy

| Endpoint | Policy | Файл |
|---|---|---|
| `lmstudio:chat` (RAG) | `chatWithPolicy` | `electron/ipc/lmstudio.ipc.ts` |
| `lmstudio:compare` | `chatWithPolicy` ×2 | same |
| `agent:start` (ReAct) | `chatWithToolsAndPolicy` | `electron/ipc/agent.ipc.ts` |
| `dataset:start-batch` (legacy) | `withPolicy` direct | `electron/dataset-generator.ts` |
| `dataset-v2:start-extraction` | `chatWithPolicy` + `externalSignal` | `electron/ipc/dataset-v2.ipc.ts::makeLlm` (ересь №1 закрыта) |

### 12.4 Watchdog LM Studio

`electron/lib/resilience/lmstudio-watchdog.ts`:

- Polling `GET /v1/models` каждые `healthPollIntervalMs` (default 15s, из preferences).
- При `healthFailThreshold` подряд (default 3) — событие `resilience:lmstudio-offline` → coordinator паузит все зарегистрированные пайплайны (extraction, dataset, forge).
- При успешном poll после offline — `resilience:lmstudio-online` → coordinator возобновляет.
- Telemetry events `lmstudio.online`, `lmstudio.offline` логируются.

Тестируется `scripts/test-watchdog.ts` с mock-сервером.

### 12.5 Coordinator (batch lifecycle)

`electron/lib/resilience/batch-coordinator.ts`:

- Pipelines регистрируются через `coordinator.registerPipeline({name, handlers})`.
- Handlers: `pause`, `resume`, `cancel`, `discard`, `flushPending`.
- `coordinator.reportBatchStart/end` — для tracking активных операций.
- `coordinator.flushAll(timeoutMs)` — graceful shutdown с лимитом времени.

Зарегистрированы 3 pipeline: `dataset` (legacy), `forge`, `extraction` (Crystallizer).

## 15. Архитектурные риски и рекомендации

### 15.1 Топ-3 риска

1. **LM Studio API breaking change** — низкая вероятность, высокий impact. Mitigation: `chatWithPolicy` retry, watchdog. **Действие:** добавить smoke `npm run test:e2e:mvp` в pre-release.
2. **Qdrant OOM при 100k+ концептов** — средняя вероятность, высокий impact. Mitigation: hard cap UI на размер коллекции; sharded collection в README. **Действие:** добавить мониторинг `qdrant.cluster()` в settings.
3. **HF rate limit** — средняя вероятность, низкий impact. Mitigation: timeout есть. **Действие:** добавить exponential backoff в `searchModels` (P2).

### 15.2 Скрытые риски

- **Scanner state global lock** при параллельном ingest >3 книг (см. §11.2).
- **HF token corruption** при прерывании записи (см. ересь №2).
- **Crystallizer без resilience** (см. ересь №1).

## 16. Контракты и инварианты

Эти инварианты проверены статически + тестами и должны сохраняться при любых будущих изменениях:

1. **Каждый внешний LLM-call → через `withPolicy`** (или `chat*WithPolicy` обёртки). ✅ полностью соблюдается с момента закрытия ереси №1.
2. **Каждое RMW-обновление shared JSON → под `withFileLock` + `writeJsonAtomic`.** **Нарушение:** HF token (см. ересь №2).
3. **Каждый IPC handler принимающий filePath или collection name → через `parseOrThrow(*Schema, ...)`.** Покрыто частично (scanner, qdrant), полное покрытие в P1.
4. **Embedder pipeline существует ровно в одном экземпляре** на процесс. ✅ через `embedder/shared.ts`.
5. **URL LM Studio и Qdrant резолвятся ровно через `getEndpoints()`**. ✅ единственная точка истины.
6. **ContextIsolation = true, nodeIntegration = false, CSP активна.** ✅ `electron/main.ts`.
7. **Все `before-quit` пути вызывают `abortAllAgents`, `abortAllBatches`, `abortAllIngests`.** ✅ обе ветки.
8. **Чанк ID детерминирован** для idempotent resume ingest. ✅ через `bookSourcePath + sectionIdx + paragraphIdx`.
